On Thu, 29 Aug 2002, Martin Ginkel wrote:

> Exactly what I meant, for example:
> - Promot has and wants modularity
> - same for Stochsim and complex species
> - VCell and spatial distribution (specialized PDE)
> - Cellerator and flexible (moving) arrays
> 
> I believe the way an *exchange* standard makes sense is first 
> to support agreed standard features and to export (translate)
> tool-internal features into these standardized elements.
> (common denominator approach)
> Of course there then is loss of information in export. But the 
> model can be used in many other tools. And the implementation 
> effort for the tools is not too high.

I'm not sure I share this view. SBML should not only be a language to 
exchange models, but also to store models. Obviously all simulators do not
need the same level of details. That does not mean we don't have to allow
for the possibility to store all the information we possess. And the
package system proposed in the document "miscellaneous features" is perfect
for that. Here is a specific example, based on StochSim (how surprising!).
We can export multistate species and reactions in SBML1 and SBML2 files (we
will eventually, I promise). We just enumerate all the possible states, and
write down all the equations with the modified parameters. 

      k
A + B -> C   A and B possesses 2 binary flags affecting the reaction

  |
  V

          k0000
A00 + B00 ----> C
          k0010
A00 + B10 ----> C
          k0001
A00 + B01 ----> C
          k0011
A00 + B11 ----> C
          k1000
A10 + B00 ----> C
          k1010
A10 + B10 ----> C
          k1001
A10 + B01 ----> C
          k1011
A10 + B11 ----> C
          k0100
A01 + B00 ----> C
          k0110
A01 + B10 ----> C
          k0101
A01 + B01 ----> C
          k0111
A01 + B11 ----> C
          k1100
A11 + B00 ----> C
          k1110
A11 + B10 ----> C
          k1101
A11 + B01 ----> C
          k1111
A11 + B11 ----> C

Now Jarnac can run the simulation. But if I import back this file in 
StochSim, how can I now that A00 and A11 are two states of the same species, 
but not  B10? I can't (Or I have to design a really complex parser, 
particularly to  treat the rates ...). So I can run the simulation, but ten 
times slower.

Now, ultimately, I would like to use a storage format for StochSim as close 
as possible to SBMLX. With a system of package, I could generate two kinds 
of files, a compact one for the multistate aware programs, and one more 
verbose for the  simulators blind to the multistates.

> IMHO it's a common failure for <**ML> standards to include 
> everything and have literally *no* tool supporting the full
> featureset (in many cases it's unknown, to which standard the 
> tool agrees.)

You're right. The most famous examples are HTML4 and CSS. But for the sake 
of stability and perennity, it was necessary to have broad standards to 
avoid the wild extensions which led to the failure of HTML3. And somehow I 
think we're in the same situation.
