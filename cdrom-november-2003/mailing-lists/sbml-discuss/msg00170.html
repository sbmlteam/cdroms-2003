On 20-May-2003 Andrew Finney wrote:
> Can you guys describe the modules concept in more
> detail for the uninitiated please?
> 
> Whats the difference between moduoles and submodels?

OK, definition of terms:

Submodel: separated part in an SBML model that contains an own namespace for 
          it's elements but does not define an interface.
          For connections to the parent model or other submodels you can
          establish links to all inner parts of the submodel by (possibly
          hierarchical) model references.

Module:   also separated part (as the submodel). But defines an accessible
          interface. The parts in the interface can be safely linked
          to other modules.
          Everything else is hidden and can be changed by the modeler
          without notice. This allows for different implementations (detail)
          or model improvements without breaking other 'applications' of the 
          module.

        Clear enougth?
