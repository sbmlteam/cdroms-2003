I tend to agree with Martin's last point
which I interpret as meaning:

Using a parameter set with a model for which
it was not developed is typically a bad idea.

This to my mind implies a parameter set refers to a specific model.

(Although SBML itself can't enforce this
a tool would probably throw errors if

a) asked to analyse a model with a parameter set that
doesn't refer to the model

b) a parameter set refers to a parameter in the referenced model
	that doesn't exist (indicates something is wrong))

yours Andrew
