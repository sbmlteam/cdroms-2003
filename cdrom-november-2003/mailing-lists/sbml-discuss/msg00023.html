Mike:

The particular example I have uses an annotation element in the
listOfModifiers element of a reaction.  For Bio Sketch Pad, the
model editing application we're developing, regulation of a
reaction is represented as a distinct node in the graph representing
the model.  Rendering information associated with the regulation node
is most convenient to attatch to the listOfModifiers elements rather
than replicating it among the the referenced species or picking a
distinguished species reference to hold the information for the group.

As for the other listOf* elements, I generalized the notion since
I can see wanting to attatch information to the group of elements
uniformly.  For example, I attatched notes elements to some of the
listOf* nodes indicating the commentary applied to the collection
as a whole rather than an individual element.  See the listOfSpecies
element as an example.

I've attatched a simple, hand generated, SBML model as an example
of how I'm using this.

<sbml:notes xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
<head>
<title>Simple SBML L2 Model</title>
</head>
<body>
<h1>A Simple Model in SBML Level 2</h1>
<p>
This is a test representation of a model in SBML Level 2.  The model
is nominally a reaction of the form:
<pre>
  2A + B -> C
</pre>
with species <code>D</code> acting as a regulating agent or modifier
in the SBML terminology.
</p>
</body>
</sbml:notes>

<listOfSpecies>
<sbml:notes xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
<head>
<title>Notes on Species Defintions</title>
</head>
<body>
<h1>Notes on the Included Species Definitions</h1>
<p>
</p>
<ul>
<li>The <code>id</code> attribute is used only within the SBML definition.
  It does not represent the displayed label of the species.  The
  <code>name</code> attribute is used for the displayed label since
  it is not used for reference purposes within the SBML model and is
  not constructed from a restricted class of characters.  If the
  <code>name</code> attribute is not supplied in the SBML depiction
  of the model, the display label in BSP will be empty.
</li>
<li>The <code>location</code> attribute used in the Bio Sketch Pad is mapped
  to the compartment definition in SBML.
</li>
<li>The <code>charge</code> and <code>initialAmount</code> attributes on
  the <code>species</code> element map directly to the corresponding attributes
  used in the species description in Bio Sketch Pad.
</li>
<li>The <code>polymerizationState</code> element is an optional element
  appearing in the species annotation namespace, XMLNS attribute ???.
  It has no content, a single attribute with the default value of
  <quote>1</quote>, and is restricted to a positive integer greater
  than 0.
</li>
</ul>
</body>
</sbml:notes>

<listOfReactions>
<sbml:notes xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
<head>
<title>Notes on Reaction Definitions</title>
</head>
<body>
<h1>Notes on Included Reaction Definitions</h1>
<p>
</p>
</body>
</sbml:notes>

<reaction id="r1" reversible="false">
  <annotations xmlns:bsp="http://bio.bbn.com/xmlns/biospice/bspreaction"
               xmlns:graph="http://bio.bbn.com/xmlns/biospice/bsplayout">
    <bsp:reaction reactionType="transformation"
                  inputEdgeArcType="line"
                  outputEdgeArcType="line" />
    <graph:layout xposition="200"
                  yposition="150" />
  </annotations>

  <listOfReactants>
    <speciesReference species="A" stoichiometry="2" denominator="1" />
    <speciesReference species="B" stoichiometry="1" denominator="1" />
  </listOfReactants>

  <listOfProducts>
    <speciesReference species="C" stoichiometry="1" denominator="1" />
  </listOfProducts>

  <listOfModifiers>
    <annotations xmlns:bsp="http://bio.bbn.com/xmlns/biospice/bspreaction"
                 xmlns:graph="http://bio.bbn.com/xmlns/biospice/bsplayout">
      <bsp:reaction endPointType="promotor"
                    inputEdgeArcType="line"
                    outputEdgeArcType="line" />
      <graph:layout xposition="200"
                    yposition="250" />
    </annotations>

    <modifierSpeciesReference species="D" />
  </listOfModifiers>
</reaction>

</listOfReactions>
