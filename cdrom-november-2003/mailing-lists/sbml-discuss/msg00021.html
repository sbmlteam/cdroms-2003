  I would like to say a few words about flux models, and optimization in
general.
  In a flux balance model, fluxes (i.e. rates of the reactions at steady
state) become the variables of the model. No concentrations or kinetic
constants are required. Solving a flux balance model typically involves
the following steps:

1) Define the reaction network. The key parameters are stoichiometric
coefficients.

2) Define lower/upper bounds for (some or all) fluxes 
(e.g.: 0 < (glucose uptake) < 4)

3) Optimize for a given objective function using LP/QP (e.g. find the set
of fluxes in the network that minimize ATP consumption). These fluxes can
be compared with steady state rates obtained with kinetic models.

The option of defining bounds for variables is potentially quite general,
as any optimization problem would involve this kind of constraint (for
example, one may model 3D structures that satisfy a given set of distance
constraints between points).
Defining bounds for variables is different than defining values of 
parameters, and it may be useful also in kinetic models. 

  I don't know whether a special format should be present for defining an
objective function, but I suggest to keep this in mind too.

> will be to answer the following question: what should be the final
> set of features to be incorporated into SBML L2?  Specifically,
> we would like decisions about the following topics:
> 
>  (a) Whether the features described in the appendix of the
>      current L2 draft should be part of the final L2, or
>      left to another SBML level.
> 
>  (b) Whether the following features, requested after the
>      publication of the L2 draft specification, should be
>      included in SBML L2:
> 
>      1. flux bounds (required for flux analysis)
>      2. optional initial species concentration/amount
> 
>  (c) Whether there are any other well-characterized features
>      that are >clearly< appropriate to introduce in Level 2.
> 
> We know that many of you will be unable to attend the forum
> on the 12th.  We therefore encourage you to send any
> comments you have on the features of Level 2 along the lines
