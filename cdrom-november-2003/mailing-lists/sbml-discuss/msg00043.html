Andrew and Martin,

> On 05-Dec-2002 Andrew Finney wrote:
> > I tend to agree with Martin's last point
> > which I interpret as meaning:
> 
> In this mail I wanted to clarify my misunderstood sentence about 
> `default values'. I don't want to define defaults in SBML, but
> the model description contains some default parameter set that 
> works (and represents some special experimental setup).
> 
> > Using a parameter set with a model for which
> > it was not developed is typically a bad idea.

Typically yes especially for small research groups, but no for large
simulation projects and research community as whole.

Some groups are already preparing high throughput determination of
enzyme properties such as kn, Kd, Km and Kcat.  In fact a group in our
institute had already started streaming such data for modeling projects.
(though the data is still preliminary in terms of amount and quality..)

They'd be happy to use the standard parameter-set document format if we
publish it as a part of SBML development.  

> Yes, but: it is useful to load the same parameter set 
> when simulating slightly different variants of the same model.
> (parameters fit in, but models vary in some places).

> I wouldn't require a strict match of model-ref in the parameter-set
> and sbml. 

Agreed.  It seems some people can earn benefits from the model-ref, but
I still think it should not require the exact match or even an existence
of the tag.  If so, I think it could be a part of meta info.


regards,
-Kouichi
