The major updates to the last proposal are:

- based upon SBML Level 2
- layout information was seperated from the model definition
- all objects are based on SBase to be able to specify metainformation

On our website you can read the document online or download a pdf 
version. We also put up the schema definition and a small sample file.

Since there are not going to be IDs on all objects in SBML Level 2, 
referencing SpeciesReferences is still somewhat awkward.

An alternative to the mode for referencing SpeciesReferences in this 
proposal would be to use the metaids  for referencing.
Actually I was kind of wondering why there was the need to have an 
additional ID on some objects. Could not the metaid be used for both 
purposes, identifying objects as well as reference point for 
metainformation?  Maybe I missed something there?
Again, we would like to get as much feedback as possible in  order to 
further improve on this layout information proposal.

Ralph Gauges

	<!--The definition of SId follows.-->
	<xsd:simpleType name="SId">

			<xsd:documentation>The type SId is used throughout SBML as the type of the 'id' attributes on model elements.</xsd:documentation>
