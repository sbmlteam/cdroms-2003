Folks

Jonathan Webb of BBN technologies has written a proposal for a layout
extension
for SBML.  (The original posting is enclosed email below.)

"A text description of the schema is available at the URL:

  http://bio.bbn.com/biospice/mdl/design/layout.html

The schema itself is available at the URL:

  http://bio.bbn.com/xmlns/bsp/bsplayout
"

Please comment on its incorporation into SBML.

organizational comments:

strictly IMHO - Mike Hucka will probably have comments

If there is consensus I don't see why we can't incorporate this
into SBML before the November meeting.  For example I can take
Jonathan's
description and extend the SBML specification as a proposed Level 2
Version 2.
We could then vote on this using Survey Monkey and/or have an SBML forum
conference call to discuss this.  I think this is a viable idea
since Jonathan's proposal is close enough to what people are actually
doing with their implementations.  Similar proposals have been 
described at SBML forums previously with reasonable consensus.  

Folks what do you think?

Alternatively people may want to see more reference implementations
first.  Perhaps this may be in place before November so we can 
start the incorporation process then.   I would argue
that the existing tools have implementations that are close enough
to Jonathan's proposal that they are effectively reference
implementations
Again I'd be interested in peoples opinions on this.

Another alternative, similar to the development of level 2, is to
develop a provisional Level 2 version 2 in the next 1 or 2 months
which will be formally frozen say mid 2004.

The aim here is to avoid incompatible formats and have some
obvious place to keep the format definition.  We don't
necessarily need to incorporate this proposal into
SBML.  It could remain an annotation for example.

technical comments:

when this incorporated into SBML the
IDREF 'component' attribute should be changed to 
an Sid attribute with an id attribute introduced on
SpeciesReference.  This is to support future
model composition.

There probably needs to a mechanism for adding extra text
to a diagram which is not taken from name/id fields.
