The units I use are expressed as screen units, roughly 92dpi. Users can
override this resolution if they wish. The reason for using a physical size
is to do with font information (which is expressed in pts) and allows the
program to preproduce the actual size the model was designed to. I think
this is what Hugh said in his email. If you have a dimension of 100 units
and a font specified at 10pt, what does the 100 mean, what size is it
relative to the font since the font is specified in actual physical units
but the dimension in no units.

Herbert
