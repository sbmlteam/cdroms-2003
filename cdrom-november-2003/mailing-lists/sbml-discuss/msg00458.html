    As the person who precipitated this followup discussion, I
have no particular insight into whether having the same name for
an element with the same name as an attribute could create problems
for any software that deals with SBML/XML files.

    But in the interest of keeping SBML as intuitively obvious as
possible, I really like the idea of naming the mutually exclusive MathML
option StoichimetryMath as was suggested, since that makes it pretty clear
how it differs from the traditional Stochiometry.  Unless that could
complicate the implementation of the mutually exclusive concept, I would
be all in favor of giving it this different name whether that avoids any
software confusion between the two things or not.

     I assume in some contexts it makes sense to have a few
SpeciesReferences with StoichiometryMath even though most in the same
model have just regular Stoichiometry, as long as any single
SpeciesReference does not present both. 

					Wayne
