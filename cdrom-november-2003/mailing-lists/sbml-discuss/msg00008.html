Yep, but more specifically I meant: the model description (not the 
SBML-Standard) defines e.g. default values for parameters 
whereas the parameter-sets for a specific simulation experiment 
may override these values.

Especially: loading a SBML-Model while introducing the different default 
values of different modeling environments (for missing numbers) is very 
dangerous.
