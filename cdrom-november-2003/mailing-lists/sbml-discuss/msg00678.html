Hi All,

     I realize the discussion of SBML modifiers is taking place in the much
larger context of their original motivation, exact meaning and appropriate
use, but I wanted to remind anyone using libsbml about the following
convenience function:

       #include "SBMLConvert.h"

       /**
        * Adds modifiers (ModifierSpeciesReferences) to the given Reaction.
        *
        * A Model is needed for context to determine the set of allowable
        * Species (see criterion 1 below).
        *
        * For each symbol in the Reaction's KineticLaw, that symbol is a
        * modifier iff:
        *
        *   1. It is defined as a Species in the Model
        *   2. It is not a Reactant or Product in this Reaction.
        */
       void SBML_addModifiersToReaction (Reaction_t *r, const Model_t *m)


     This function works for any reaction with its reactants, products
and KineticLaw (in either infix or MathML form) set and is called
automatically by SBML_convertToL2() and SBML_convertReactionsInModelToL2().

     It's handy if you want ensure that you conform to L2 but don't use
modifiers in your own simulators or other programs.


Ben
