Martin,


> On 22-May-2003 Kouichi Takahashi wrote:
> > (1) In reality it is a natural number in the cell. It is exact, and is better to 
> >     support 'exact' methods such as Gillespie. 
> > (2) 'substance' is the primary representation, and concentration is secondary.
> >     This can be understood by the fact that the simplest way to have support 
> >     for changing volume is to store the value as 'substance', not 
> >     'concentration'.
> > 
> > The input/output of the simulator can either be concentration or the exact number, 
> > but the better internal representation by multi-compartment simulation is not
> > concentration.    
> 
> The argument was from a software engineering point of view:
> Species are defined in Items/Moles but whenever their id appears in some
> ratelaw/rule it is implicitly treated as concentration, which is not
> obvious and self-documenting. I consider this as the main point.

Then it is not an issue of the initial value.  
Rules should explicitly indicate it is concentration or not.
Species may, for example, have concentration property apart from the exact value. 
In E-Cell concentration property is read-only, and the core accepts only the value, not
concentration.   Its frontend allows users to input concentration, and converts it to
value when it passes to the core.   

Anyways if SBML has the compartment volume (or size, whatever), it does not make much 
sense to have concentration as the default.


> > Considering SBML as a long-term standard, I believe it is going for exact 
> > value from concentration, and it is better not to use concentration as the 
> > default.
> 
> I am not sure whether pretending to know the exact number of glucose-molecules
> in a model of a cell is really an advantage ...
> Don't trust the digits far right of the decimal point ;-)

It is an advantage, at least computationally.   
It is necessary to mix continuous and discrete simulation algorithms.   
We are currently going to implement a per-process (per-reaction) dynamic migration
between algorithms modules according to some conditions such as the absolute number of
molecules.   For example, if the number of molecules is very small (say less than ten),
ODE simulation is no longer reliable.  We should instead use some discrete and stochastic
one such as Gillespie.    If it is represented as concentration, it is much difficult to design
this.   To make it simpler it allows real number values (such as 1.5 molecules).   


-Kouichi
