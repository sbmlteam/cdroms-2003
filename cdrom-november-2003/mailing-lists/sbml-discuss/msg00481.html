> To get back to your email, Ed, Zeigler et al. distinguish 3 types
> (formalisms) of dynamic model specification (as opposed to e.g. object
> modelling, in which there is no concept of time): the Discrete Time
> (DTSS), Differential Equation (DESS), and Discrete Event (DEVS) System
> Specifications. DESS includes ODE and PDE descriptions. DTSS includes
> things such as cellular automata (which are updated as a whole in a unit
> 'time' step). I think (but I'm not entirely sure) that DEVS includes
> stochastic models (in which possible state transitions have certain
> probabilities), but also threshold models (in which a state transition
> occurs when a certain parameter crosses a given threshold). Have I
> understood this correctly?

> > > > > -----------------------------------------------
> > > > > Charter of the SBML Hybrid Models Working Group
> > > > > -----------------------------------------------
> > > > >
> > > > > Background
> > > > >
> > > > > Hybrid Models are those that include phenomena modeled in
> > > > > fundamentally different ways or which include elements
> > that change
> > > > > their mode of operation, or both.  For example, discrete and
> > > > > continuous models are fundamentally different.  A model
> > > > including both
> > > > > would be hybrid as would a system including both non-spatial
> > > > > components (represented by ODEs) and spatial components (PDE's,
> > > > > reaction-diffusion).
> > > > >
> > > > >
> > > > > Mission Statement and Scope
> > > > >
> > > > > Except for combined continuous/discrete simulation,
> > little hybrid
> > > > > modeling is being pursued in the SBML community [is
> > this correct?].
> > > > > Thus the Hybrid modeling group is at an earlier stage than
> > > > the other
> > > > > groups and the mission statement reflects this.  Our
> > > > mission statement
> > > > > is:
> > > > >
> > > > >    1. Enumerate 5 sample problems ("challenges") to be
> > > > addressed by hybrid
> > > > >       approaches.
> > > > >    2. Determine what is needed from SBML to express
> > these models.
> > > > >    3. Determine what, if any, deliverable there is for
> > the Level 3
> > > > > deadline.
> > > > >
> > > > > Deliverables:
> > > > >    -  proposal for SBML L3 extensions (this is not
> > expected to be
> > > > >       support of hybrid systems, but only support of
> > > > limited capabilities
> > > > >       like Events done with an eye toward future, fuller
> > > > hybrid systems).
> > > > >    -  challenge problems
> > > > >    -  SBML requirements for Hybrid systems modeling.
> > > > >
> > > > > [This is not meant to be a waterfall process.  I am
> > > > actually probing
> > > > > here to see if others feel as I do that we are at the very
> > > > beginning.
> > > > > If so, then some time on seeing if we are all talking about
> > > > the same
> > > > > things when we say the same words is worthwhile.  The
> > > > requirements and
> > > > > enumeration of challenges is thus more agreement on a
> > > > destination than
