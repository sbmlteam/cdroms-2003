For some time now, I've been pushing to get
diagram markup out of the <model> section
and into a separate section, say, <view>.
First of all, it offends my more sensitive
"model-view-controller" feelings ;) but
since my first discussion with Herbert about
this, it became clear to me that this had to
happen for a much more practical reason:
embedding diagram data inside the model
implicitly assumed that there could only
be one diagram. Since many of the other
developers (that is, in addition to the
Virtual Cell group) need to have multiple
diagrams, it seems clear that this separation
must happen. Here's the (very rough) idea:
<sbml>
	<model> ... </model>
	<view>
		<diagram width="17" height="5" name="OverallDiagram">
		...
		</diagram>
		<diagram width="5" height="5" name="Cytosol">
		...
		</diagram>
		<diagram width=3" height="8" name="PlasmaMembrane">
		...
		</diagram>
	</view>
</sbml>

Also, there might need to be some kind of
indication of a "default" diagram, one that
an importing tool that only displays one
diagram should choose (that is, if you can
only display one diagram, this is the one
to pick).

John
