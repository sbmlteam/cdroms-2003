I see what you're saying, however there appear to be two different
architechures in use in the simulation community, one uses local names
spaces and another uses global names spaces. I happen to use global names
spaces mainly because I've worked with script type interfaces over the years
hence it makes more sense to me. In the example you give I would simply use
the same name in the different rate laws, this is in fact what users do with
when writing the scripts.

Someone mentioned they have a converter to convert from one form to another,
this might be useful. I guess the long term solution is for all simulators
to use the same data architechure.
