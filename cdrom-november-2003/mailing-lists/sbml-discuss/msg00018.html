Hi Andrew and Martin,

I'll join the fray by agreeing with both of you.  If we
have models with lots of parameters, it would be nice to
make some or all of them have default parameters.  Why not
simply allow users to assign default initial values to a
user-selected set of parameters?  The user can then ignore
them as convenient (perhaps we wish to assume room temperature
most of the time but don't want to always list it), but give
values when appropriate.  For parameters that should always
be given, don't assign a default and the user will be forced
to supply one.  We may wish to make users always supply initial
values for species.

This is really a question of language design philosophy and
usage models.  For machine generated models it won't matter.
For user written models, there is a tradeoff between adding
the convenience of default values thus not requiring every
parameter to be listed every time as opposed to defaults
that are occasionally forgotten.

I will say voice the opinion that results for a model experiment/simulation
are useless without the input parameters as well.  We can
debate if it makes more sense to associated the input values with
the model or with the experiment results.  IMHO this would probably
best be managed as a tool issue, although a standard format for a
configuration file would be nice to make models portable.

Greg
