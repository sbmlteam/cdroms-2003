Mike & Andrew,

I support the analysis you made and therefore the conclusion. 

This discussion brings up an issue that has been dormant (but here!) since the 
freeze of level 1 version 1: the need for a longer period of time between 
deciding on features and actually freezing (publishing) the specification. When 
I and Herbert were working on making existing software import/export SBML level 
1 we discovered a number of features that could have been decided in a better 
way, and others that were missing. While you made a very good analysis of the 
problem of default compartments, it is easy to miss out important consequences 
from such analyses. I propose that the next specs be accepted first as draft 
for a considerable period of time (say 1 year), and then adjusted according to 
feedback from those who have been implementing for the draft. It would also be 
important to publish reference software that demonstrates use of the standard. 
This procedure would ensure that we iron out problems with the standards. Given 
that we already have a standard for the most common features of models, we 
should be careful to get the standard right for the features that are only now 
being thought of (ie most simulators do not implement them yet, in some cases 
none)
