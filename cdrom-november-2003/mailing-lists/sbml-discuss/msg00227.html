> Once you think of composing a model of several submodels, you will find 
> that it is much easier to compose a model from submodels where initial 
> concentrations are specified than from submodels where initial amounts 
> are specified: A submodel that uses concentrations can be included in a 
> composite model with a different volume without change, but a submodel 
> that uses amounts has to be copied and modified, or EACH species has to 
> be overridden by the composed model to get the initial values right.
> 
> Remember, SBML is NOT about the internal representation, but about the 
> easy exchange of models, and eventually about the reuse of models.
> 

> Jörg Weimar.
> 

So if I understand you right. If I change the volume of a sub model I -- ALWAYS -- want to keep the concentration and not the particle number. If this is the case you are right. 

But I strongly believe this is will not always be the case.  We can not assume either concentration = const or amount = const. Therfore, the COMP group has to define an interface to a model specifying what has to be done in that case of a volume change. 

I am not against change but I need to see a gain and I completely miss that here.
