	Sven>This could be a sequence of curves (I mean the curves that 
	Sven>are descibed by start point and end point and two additional points that 
	Sven>define the tangents to these points, I believe they are called bezier, but 
	Sven>I´m not shure about the terminology). This seems to be the way drawing 
	Sven>programs like adobe illustrator (or kontour) work. 
	 
	This is exactly what Jdesigner does, I took my cue from applications like illustrator. The calculations required to get things looking nice are quite tedious though. There are a bunch of linear equations that need solving in order to get the bezier segments to blend in a nice way, essentially the end tangents must match. Each time a control point is moved, the tangents have to be recomputed. If you look at the Jdesigner annoation spec you'll see that I store the set of control points and center point for each reaction.
	 
	I think we should at least permit straight line segements as there would be very little work in renderning these. Beziers are another matter, however if a app cannot render the beziers they can simply draw straight line segments and they can use the centre point or compute the centroid.

	>The first way (different curve types) would be easy to write, since all 
	>programs could write just like they internally represent the curves, but 
	>might involve conversion calculations while reading (if the curves are given 
	>as a curve type that my program doesn´t use internally). 
	>The other way (just one general curve type) would be easy to read (at least 
	>only one conversion routine is needed) while it might be difficult to 
	>translate the internally used curves to the one standard curve type in the 
	>sbml output.
