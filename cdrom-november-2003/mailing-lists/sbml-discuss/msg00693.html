Stefan, I don't think any one is saying that the list of modifiers is 
redundant (I'm sure someone will think of something to use it for), all 
that's being asked is guidance on how one should generate the list of 
modifiers. I think we can all agree that it is too difficult to generate 
automatically from the rate law and it will require the user to 
specify the list explicitly during model contruction. Most software, 
including Gepasi as far as I am aware (I just tried it out) will not be able to generate 
the correct list of modifiers. If it is agreed to include reactants 
and products in the list of modifiers then we'll all have to modify our 
software, this also includes Ben's C library. Presumably Copasi is set up 
to  support the list of modifiers?


Herbert


On Tue, 14 Oct 2003, Stefan Hoops wrote:

> Hello All,
> 
> I am watching the thread for a while and can't withstand to put my two cents in :).
> 
> The list of modifiers and the rate law describe two different things. The list of modifiers is a conceptual thing whereas the rate law is describing a very specific implementation of a kinetic law compliant with the concept described though substrates, products, modifiers, and if you want parameters.
> 
> We can have different kinetics for the same concept and as such neither the list of modifiers nor the rate law is redundant. 
> 
> I do not see how it is possible to get the list of modifiers from the kinetics in the case described below by Upi. We need it! 
> 
> I agree that it is possible to assign a rate law which does not comply with the concept described by substrate, product and modifiers. Whether SBML should allow or forbid this should be the point of discussion but not whether we need a list of modifiers or make it optional. 
> 
> SBML shall facilitate the exchange of models especially reaction based. Let me give an example how I think SBML can be used. We have a set of reactions. The described reactions may have observed modifiers but no known rate laws, so it would make perfect sense to me to write a list of modifiers and skip the rate law. Other people can take the model and add compliant rate laws to the reactions and play with it. This process is what SBML should enable. 
> 
> Thanks!
