Hi Kouichi,

> As we discussed a bit in the previous meeting, we would like to help
> something to wrap it for Python.

     That's great news!  I'm more than happy to receive help and
contributions from anyone who is willing.  I recall talking to
Takeshi Sakurada at the last SBML/W workshop.  Is he the developer
with whom I should get in touch?

> We have a lot of experience in Python C API and Boost Python Library
> (C++ <-> Python interconnecting).

     I'm curious about E-Cell's experiences with Boost.  I know of it,
but have never used it myself.  I designed libsbml from the beginning
so that it could be wrapped by other languages.  In fact, my design
and naming conventions were partly influenced by the recommendations
of the Simplified Wrapper Interface Generator (SWIG; http://www.swig.org)
project.  My hope is that SWIG will be viable for libsbml, as it supports
many language targets:

       - Tcl
       - Python
       - Perl
       - Java
       - Ruby
       - Mzscheme
       - Guile
       - PHP

     I know however that Boost is more advanced than SWIG in some areas,
e.g., C-based shadow classes.  All of this will make for interesting
discussions and investigations and I look forward to the wrapping
expertise of your group.

     Just to let everyone know, I'll be setting-up a libsbml mailing
list on SourceForge soon, so as not to clutter these lists with
tangential-topic posts.

     Thanks.


Ben
