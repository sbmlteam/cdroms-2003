In most programming languages the distinction between 1.0 ( a float 
constant) and 1 ( an integer constant) is also made, but it is not 
usually necessary to specify separately, since it can be infereed easily 
from the presence of the decimal point. Similarly, if denominator is 
present, then the stoichiometry if fractional. No need to specify this 
separately.

Another issue is the question whether a model uses ANY fractional 
stoichiometries, or whether a simulator is capable of handling them. 
This is a question that refers to the broader issue of "capabilities", 
which we will discuss in the "controlled vocabularies working group".
It is a much braoder issue, since many other features of SBML may or may 
not be used in a model, and may or may not be supported by a tool.

My opinion on stoichiometry: There is no need to specify separately the 
type used, and having stoichiometry:double and denominator:integer is 
not a contradiction. Recall that small integers are represented exactly 
by floating point numbers, while small fractions, such as 1/3 are not 
represented exactly, neither in decimal notation (0.333.... how many 
digits) nor in binary notation used for floating point  numbers 
internally. to represent 1/3, we need
stoichiometry="1" denominator="3", while for measured stoichiometries we 
need stoichiometry="0.1347" or so.
