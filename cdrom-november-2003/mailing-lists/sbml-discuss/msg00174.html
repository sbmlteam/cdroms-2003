Charter of the SBML Complex Species Working Group
-------------------------------------------------

Background

At the recent SBML forum meeting in Florida it was decided to setup
working groups to develop
proposals for incorporation into SBML.  Several people at the meeting
signed up to a working
group to develop a proposal for complex species in SBML

Mission Statement and Scope

Definition of an extension to SBML to allow the construction
of models that contain species constructed from components where each
component may have
several states.  These species are called complex species.
In SBML L2 each combination of component states has to be enumerated as
a separate species.
An example of this type of model is that described in
B. Goldstein et al. Molecular Immunology 1137 (2002) 1-7.

The aim is to create an encoding of models containing complex species
which doesn't require
a combinatorial enumeration of states.  This encoding will closely
resemble the form in which
a modeler would describe a model.

Any viable proposal must:

- allow species to have multiple states

- allow species to be composed from multiple components where each
component may have multiple states

- allow generalized reactions to be defined: this requires that sets of
species states can be defined
	to be the reactants, products and modifiers of a reaction (as
opposed to a specific species
	state as in SBML L2).  For example the state of part of a
reactant can be undefined or "wildcard"
	indicating that that part of the reactant doesn't affect a given
reaction.

- consider how the proposal interacts with other proposals for Level 3
including model composition and arrays.

Termination criterion:

   Complex Species is included into SBML Level 3

   or

   The idea is considered as nonsense for Systems-Biology and dropped
   from the SBML Agenda

   Timeframe: I expect a working proposal and first reference
   implementations for the a SBML Forum Meeting in 2004

Deliverables:

   - a Complex Species proposal for incorporation into SBML L3
   - a reference implementation that can read and write the proposed
standard including L2 in a
     useful software context
   - at least one published model encoded in the proposal
   - optional: a tool that can translate from the proposed standard
including SBML L2 to
     unextended SBML L2.

Communications will be via sbml-discuss.
