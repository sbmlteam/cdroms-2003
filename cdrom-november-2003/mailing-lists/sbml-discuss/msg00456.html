Having reread the XML spec and the infoset spec, there's nothing
in the well-formedness constraints connecting the names of attributes
and the names of child elements.  At this level, there is nothing
preventing the an element having an attribute with the same name as
a child element.

The XML Schema spec is a bit more to go through but a quick read
doesn't turn up a restriction on both appearing.
