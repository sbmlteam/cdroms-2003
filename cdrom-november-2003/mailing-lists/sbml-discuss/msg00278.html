Mathematica has a function Rationalize, which (quoting from the help text)

Rationalize[x] takes Real numbers in x that are close to rationals, and 
converts them to exact Rational numbers.

Rationalize[x, dx] performs the conversion whenever the error made is 
smaller in magnitude than dx.

Example: Rationalize[3.78] -> 189/50

A rational number p/q is considered "sufficiently close" to a Real x if

|p/q - x| < c/q^2 , where c is chosen to be 10^-4.

Rationalize[x, 0] converts any x to rational form.


End quote


I imagine any software package doing rational arithmetic could perform 
this operation when reading in stoichiometries that are floats. All the 
other (actually most) programs/developers either support only integers 
anyway or use flotaing point. For them, life would get easier, since 
they are not concerned with rational arithmetic, and they do not need to 
program this function just to create valid SBML.

> There is a whole class of analyses based on stoichiometry alone. The
> operations involve a lot of matrix manipulation, particularly elementary row
> operations. For large systems >100+, it's far better to use rational
> arithmetic in the math rather than floating point numbers because of the
> propagation of rounding error. For large systems it is therefore not
> possible to be 100% sure that the conservation vectors for example are
> actually the correct set without doing so post analysis checking which I'm
> sure most simulator tools don't do. These vectors need to be computed for
> people doing dynamic analysis as well so the problem is not just confined to
> stoichiometry analysis.
> 
> I think Mike'd idea for alowing either float or rational sounds like a good
> idea, keeps everyone happy.
