Dear SBML fellows,

In the proposal of Andrew et al. the Features, i.e. the required packages,
are listed in parallel with the models. It would perhaps be a good idea to
include it _within_ the model. In such a case, an SBML file could contain
different models describing a system with more or less details. The
simulation tool would select a model suitable for it, if such a model
exists. This would require a way to set-up a priority for the models, in
case several of them are suitable for the current tool (for instance the
model the more detailled, i.e. with the more features, the better?)

<sbml xmlns=" ...
    <model name="multistate">
        <listOfFeatures>
	    <multistate/>          <-- suitable for StochSim
	</listOfFeatures>
	...
    </model>
    <model name="singlestate">
	...                        <-- suitable for Gepasi or Jarnac
    </model>
</sbml>

Any opinion?
