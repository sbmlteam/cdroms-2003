A minor proposal: boundaryCondition attribute shouldn't determine
variable is constant
========================================================================
==============

The species element has an boolean attribute boundaryCondition which is
false by default.  In Level 1 a truth value indicated that the species
concentration was fixed for the duration of the simulation.

Herbert Sauro has proposed that in level 2, and I agree, that the truth
value should indicate that the species is not effected by reaction
elements.
The species is thus 'on the boundary of' the system of reactions defined
by the set of reaction elements.  

In the absence of rules referring to the species, the species will be
constant.

A Wider problem: determining when a variable is constant or not
===============================================================

This leads us to a bigger issue: when can a variable (species amount,
compartment volume, parameter value) vary during a simulation.
In the absence of algebraic rules this is straight forward to
determine: the variable is constant unless there is a rule
(must be assignment type) that determines how the variable
varies.  

Background: assignment type is x = f(P) or dx/dt = f(P)
algebraic type is  0 = f(P) (P is the set of variables (with some
restrictions)).

A reaction is roughly equivalent to a dx/dt rule for each product and
reactant.

If we introduce algebraic type rules we have a problem:
the model may intend some variables be determined by the
rule and some variables to be constant for the duration
of the simulation. 

for example given 
k = 0.02;
x = 0;
y = 1;

dy/dt = f();

0 = (k * x + y) - 1

In SBML there is no way to specify that k is constant and that x can
vary
during the simulation.

A Proposal: solution to above problem
======================================

Add a 'constant' boolean attribute to the 'species', 'compartment' and
'parameter' elements.  In the case of species this attribute defaults to
false.  In the case compartment and parameter this attribute defaults to
true.

It is invalid to set 'constant' to false on local 'parameter' elements
(since no rules can affect the parameter variable).
It is invalid to create assignment rules for variables that are
constant.  It is invalid have a model in which a species
concentration can vary through reactions and to define it
constant i.e. the species is not a boundary condition and occurs
as a product or reactant.

Comments
========

Please comment on the above - feel free to rubbish it!
