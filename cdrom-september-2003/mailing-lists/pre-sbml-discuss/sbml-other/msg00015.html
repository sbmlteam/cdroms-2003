> what happens if the given initial
> conditions contradict the algebraic constraints
> e.g.
> x = 0
> y = 1
> 0 = x + y

The solver stops with an error ;-)
The point is, if you do DAE then the initial conditions have to be 
consistent. In serious simulators before simulating anything there is a
numeric equation solver invoked that tries to find a set of initial 
values that fulfills the initial conditions starting with the given values and
modifying the states.
It solves the values of dotx and x. In some tools it is possible to 
provide additional equations for this process. (Since dotx are also unknowns
you may have size(dotx) more equations).
There are several publications, dealing with this problem of finding consistent
 conditions e.g.:
Pantelides C.C. The consistent initialization of differential-algebraic systems.
SIAM J. Sci. Stat. Computing. 9 213-231 (1988)

> 2nd consider the actual simulation

Integration can only start with good initial values.

> should variables e.g. parameters or boundary
> condition species that would be constant
> in the absence of algebraic equations
> vary to satisfy algebraic constraints?

No. Unless you have equations to define them. (Input functions for
boundaries, parameterRules).

> 
> exists for the  variable.  If we put algebraic rules into
> the mix I think we have a problem:
> what variables are constant and what can vary 
> to satisfy constraints?

The states can vary: species, compartments, state-like parameters.
If this doesn't lead to success, then the initvalues must be defined better.

> am I being paranoid?
