Title: Minimal SBML Proposal comment
Regarding encoding arbitrary symbols in SName I would object to use the of the underscore character as a delimiter. 

(1) In particular, the statement "If a program cannot display the character, it should
leave it as-is." is objectionable because some programs can not easily display the underscore character - in Mathematica, for example, the underscore is reserved for pattern matching and can not be embedding in a symbol. The only way to do this would be to convert the symbol to a string, which has a different meaning in Mathematica. I would suggest rewriting this sentence as "If a program cannot display the character, it should be displayed in a program-compatible form as closely resembling the SBML syntax as possible."

(2) In place of the underscore might it be better to retain the XML form "&#xxxx;" and allow use of XML entity statement, e.g.,

<ENTITY kappa "&#03BA;">

and then the SBML would use "I&kappa;B" for the example in the text.

(This is also difficult in Mathematica but is more familiar)
