Victoria Gor emailed me the following....

To refresh your memory, we were thinking that the best way to introduce
arrays and graphs in SBML2 is through examples. So, the first example I took
is the SBML model produced by Bruce Shapiro's cellerator (it is attached to
this message). I renamed some elements and introduced some new ones (since
we
are using a compact array notation now), but the model described should be
the
same. Of course, SBML2 representation below is by far not unique, and I
would
have to do the same procedure for a lot more examples in order to finalize
SBML2 structures and may be even restate them in more elegant ways. But for
now, I would be happy with any comments and directions you have for me.
I numbered some lines in this example so I can refer to them. Now let's
list some of "new"(SBML2) features provided by this example.

0) "range" element as in lines 17. and 18. provides the range for specified
in "name"
attribute variables. Listed in "", such variable names refer to an entire
range as in
3., 7., 8. If such variable names are listed without "", as in 11 it means
that range
variable name would have to be immediately expanded by
interpreting program (could be program that translates SBML2 into SBML or
could be a
direct parser of SBML2). In any case all the unquoted references to range
name should
be expanded first for all possible values of the range.

1) "[]" notation. It means that the element with name containing "[]" is an
array.
If no index is specified inside the brackets, that means an entire array is
used.
"dimension" element specifies the dimension of the array by referring to
pre-defined
ranges.   If there is a ":" in one of the "[]" as in line 11., it means that
given variable is a vector and vector operations should be applied to it.
":" in two
dimensions implies matrices and matrix operations.

2) "instance" inside the array element is used to assign values to
attributes of
individual members of the array.

3) "connection" (line 6.) is an element that describes a set of links (as we
discussed
previously). "function_based" connection will contain a "mapping"  element,
whose attribute:
"formula" is a boolean formula which is true when connection exist, and
false otherwise.
To process connections, interpreting program might create both a binary map
(sparse matrix)
which indicates which connections exist and lists of indexes for each array
index indicating
to what array elements given array element is pointing. So 9x9 map will be
created for
connections ngb[][] (line 6.) and 9 lists pointing to different number of
array elements,
representing adjacent neighbors in given example.

4) "compartmentReference" in line 9. specifies an array of arrays and it
means that every
cell in cell array contains a 3-dimensional array of species s[].

5) "connectionParameterRule" is an element that assigns an attribute to
every valid
connection. That is why "connectionReference = "ngb[][]"" is used. By valid
connection
I mean the value of 1 in the ngb map of interpreting program.

6) "sigmoid()" is a predefined function that should be added to built-in
functions for SBML.

7) "sum{}" should be a built in function in the interpreting program. It
means that such function
should be expanded immediately after all the range variables are expanded.
This function
passes two string parameters. First parameter indicates what is a summing
variable and it's
range. So, in line 13. after expansion we will have "L in ngb[1][L]"... "L
in ngb[9][L]",
which will mean that "L is a set of neighbors of cell[1] ... L is a set of
neighbors of
cell[9]". the second string is a formula of L which would have to be
evaluated for each L
in the set and then summed.
