Rowan Christmas wrote:
> I have only implemented straight lines and bezier curves, so I would 
> like to see an option that does not require me to use splines ;) I think 
> that a line-type field is needed, and that should determine what the 
> list of points means ( i.e. straight lines, it is a list of points; 
> splines, it is a list of quads; etc. ).  Presumably people would 
> populate the list accordingly, and umimplemented types would just get 
> drawn in a default way that connects the two nodes.
> 
> --rowan
> 
> Herbert Sauro wrote:
> >	As you probably know, I use bezier curves in jdesigner. For
> >	uniuni reaction I use just one bezier, for other reactions as
> >	use additional beziers, thus a unibi reaction uses one bzier
> >	from the uni side and two beziers to the bi side. This means
> >	that I acquire a centre point and to make things look nice the
> >	slopes at the end of the intersecting beziers match. I've
> >	thought about using other types of curve but never got round
> >	to it. As for sbml strorage, I store all control points and
> >	the centre point. Although one could compute the centre based
> >	on the centroid between all the nodes, I allow the user to
> >	move the center point, it make quite a bit of different to
> >	what users can draw.
> >	If there are going to be different ways to draw the edges then
> >	we will proably need some information indicating what type of
> >	edge drawing it is, eg splines, beziers, straight edges with
> >	kinks etc.
