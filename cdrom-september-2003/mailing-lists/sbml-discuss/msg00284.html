> Mathematica has a function Rationalize, which (quoting from 
> the help text)
> 
> Rationalize[x] takes Real numbers in x that are close to 
> rationals, and 
> converts them to exact Rational numbers.
> 
> Rationalize[x, dx] performs the conversion whenever the error made is 
> smaller in magnitude than dx.
> 
> Example: Rationalize[3.78] -> 189/50
> 
> A rational number p/q is considered "sufficiently close" to a 
> Real x if
> 
> |p/q - x| < c/q^2 , where c is chosen to be 10^-4.
> 
> Rationalize[x, 0] converts any x to rational form.
> 
> 
> End quote
> 
> 
> I imagine any software package doing rational arithmetic 
> could perform 
> this operation when reading in stoichiometries that are 
> floats. All the 
> other (actually most) programs/developers either support only 
> integers 
> anyway or use flotaing point. For them, life would get easier, since 
> they are not concerned with rational arithmetic, and they do 
> not need to 
> program this function just to create valid SBML.
> 
> Jörg Weimar.
