Many thanks for your comments.
Sorry I have taken a while to comment on them.
I'm now working through them incorporating them into
the document.

> Page 17:
> 
> section 4.5.3
> Add:
> 
> The fact that size does not have a default value means that a model 
> which contains rate laws must have a size value for the 
> compartment of 
> each species that appears in a rate law.
> 

We decided at the forum in December in Stockholm
that all initial conditions can be left out from
a model: they can be supplied from another data source
seperate from the model eg parameter sets.  Also
having missing data allows a modeller to define precisely
what is and isn't know about a given model.

> Section 4.6.4:
> 
> I think that the units of a species should always refer to substance 
> units, not concentration units. The reason is that in the way it is 
> specified now, there is a crossdependence of the presence of one 
> attribute to the meaning of another attribute. Another 
> problem appears 
> if neither initialAmount nor initialConcentration is given. This can 
> legally happen in a model where some species are set by rules.

I have adopted this change and made the required changes in the text.

> Page 21, Section 4.7.3:
> Should parameters have default "dimensionless" units?
> 
> The default units for parameters are "dimensionless".
> 

These units are left unspecified when not given.  (Parameter units can
be 
determined manually from forumlae.)  In most cases
they won't be dimensionless.  If the modeller doesn't
supply the units we shouldn't infer anything: its a weakness
in the model.  But the model can still be simulated.

> Page 26 Section 4.9.1
