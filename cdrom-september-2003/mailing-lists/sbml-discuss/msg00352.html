  JWagner> I guess the way I was thinking about the whole
  JWagner> math display is...  Either a program wants to
  JWagner> display math in a nice and formatted manner, or
  JWagner> it doesn't. Those that don't want to display
  JWagner> formatted math simply won't display formatted
  JWagner> math. But, those that *do* want to display
  JWagner> formatted math are faced with doing the drawing
  JWagner> and layout themselves. For *those* applications,
  JWagner> it would be very helpful to have the MathML,
  JWagner> because it basically specifies the layout for
  JWagner> you.
 
  JWagner> For example, in the Virtual Cell, when you type
  JWagner> in an expression like (a+b)/(c+d/e), Jim Schaff
  JWagner> has written a very nice algorithm that takes the
  JWagner> expression tree as input and outputs a "format
  JWagner> tree" very similar to the MathML display.
 
  JWagner> So I was thinking it might make it easier for
  JWagner> those people wanting to do something like that.

I agree with you about the MathML presentation.
