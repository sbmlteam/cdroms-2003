I would tend to make the following interpretation:

 - Models identify (generally) labeled parameters within their
   definition.

 - Model definitions exist without reference to specific _numeric_
   values for at least some of their parameters.  Whether this is
   because the model developer does not know definitive values
   for these parameters or because the parameter may take on a
   range of values depending on various conditions I propose is
   less relevant for the current discussion.

 - Collections of parameters are meaningless without (at least)
   reference to the associated model.

 - There may be many collections of parameters associated with
   different _uses_ of the same model.  These collections may
   evolve over time without change to the underlying, referenced
   model.

This suggests to me the parameter sets should be distinct in some
way from the definition of the model but should never be defined
without reference to a model (and probably other information as well;
how the parameters are used or generated as examples).
