Mike:

The particular example I have uses an annotation element in the
listOfModifiers element of a reaction.  For Bio Sketch Pad, the
model editing application we're developing, regulation of a
reaction is represented as a distinct node in the graph representing
the model.  Rendering information associated with the regulation node
is most convenient to attatch to the listOfModifiers elements rather
than replicating it among the the referenced species or picking a
distinguished species reference to hold the information for the group.

As for the other listOf* elements, I generalized the notion since
I can see wanting to attatch information to the group of elements
uniformly.  For example, I attatched notes elements to some of the
listOf* nodes indicating the commentary applied to the collection
as a whole rather than an individual element.  See the listOfSpecies
element as an example.

I've attatched a simple, hand generated, SBML model as an example
of how I'm using this.
