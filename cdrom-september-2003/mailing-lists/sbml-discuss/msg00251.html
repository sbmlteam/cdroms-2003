> If you need to keep references from an SBML model to high-quality
> rendering, better in my opinion to use URLs to allow SBML to support
> various graphics formats:
> <diagram-renderings>
>  <diagram href="http://server1.edu/sb/models/rendering1.gif"/>
>  <diagram href="http://server1.edu/sb/models/rendering1.svg"/>
>  <diagram href="http://server1.edu/sb/models/rendering1.pdf"/>
> </diagram-renderings>
Hey, that's a pretty good idea...we could use this idea...

You know how the <IMG> tag in HTML has an ALT="..." that
specifies a textual replacement for those browsers who can't
display images? What if we had something like,

<diagram name="SomeName" ... alt="http://server1.edu/sb/models/rendering1.gif">

that could serve more or less the same function? That
is, if you had a tool that didn't do any kind of
layout, it could very easily display the alt image
instead. We (the Virtual Cell) might use it when we
import a model, and there might be a piece (maybe
a membrane) that we couldn't do the layout on...we
might display that image in a window and tell the
user to layout the crap in the membrane by hand (based
on the image).
