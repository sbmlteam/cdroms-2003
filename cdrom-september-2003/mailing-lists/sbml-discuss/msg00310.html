You mean, why are they singular names rather than plural?

Here is an explanation for why the current form makes sense.
Consider some variable that has, as its value, an
array/collection of some data.  To make this concrete,
suppose that each data item is an integer and we're using
Java, and the variable's root name is 'poster'.  There are
two schools of thought in how you might name the variable.

      int poster[5];
or
      int posters[5];

If you choose the former, statements like the following
make grammatical "sense":

    for (int i = 0; poster[i] < 0; i++) ...

The notion is simply that whenever you refer to an
individual in the array, you're speaking of "poster i",
not "posters i".

On the other hand, the plural form ("posters") makes more
sense when you speak of the whole collection/array, in
sentences like "here I'm going to assign the 'posters' ...."

If you'd like, I can go find examples of software that uses
one approach and other software that uses the other.

Now, personally, I've gone back and forth myself, and can't
seem to make up my mind, but lately I've been going with
the plural form (i.e., the version you favor).

Do we need to put this to a vote?  I don't personally care
which form we use.
