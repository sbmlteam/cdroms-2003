Dear Andrew,

here my comments on SBML L2 V1 Dfinal (up to page 29 so far):

Page 12:

Figure 4 still has too many ii's in version: positiiveInteger.

I still suggest exchanging "version" and  "level" to "level" and 
"version" in the second but last paragraph of section 4.1, and also 
replacing "1" and "2" by "2" and "1", respectively.

Page 15:

Other examples for unit definitions might be "billions of molecules per 
minute" (gmolekpermin):

<unitDefinition id="gmolekpermin">
     <listOfUnits>
         <unit kind="item" scale="9"/>
         <unit kind="second"
		multiplier="0.01666666666666666666666" exponent="-1"/>
     </listOfUnits>
</unitDefinition>

Temperature in Fahrenheit:

<unitDefinition id="fahrenheit">
     <listOfUnits>
         <unit kind="celsius" multiplier="1.8" offset="32"/>
     </listOfUnits>
</unitDefinition>


Page 15, last paragraph:

Unit consistency should be preserved.

Delete the comment about using only multiplier and scale.

Comment: for redefinition of area, one needs an exponent 2 for a length 
unit, for redefinition of volume, one could use metre^3 as a basis for 
scaling instead of litre.


Page 17:

section 4.5.3
Add:

The fact that size does not have a default value means that a model 
which contains rate laws must have a size value for the compartment of 
each species that appears in a rate law.


Section 4.5.5

After "being the initial size of the compartment",
add
", which should be the same as the value specified by the rule at time 
0". (Or leave it as is, see Section 4.8.4, where it is stated that 
initialValues should be ignored).


Page 19:

Section 4.6.3: insert "set by a rule" to give

" unkown, not required for analysis, set by a rule, or available from an 
external data source."


Section 4.6.4:

I think that the units of a species should always refer to substance 
units, not concentration units. The reason is that in the way it is 
specified now, there is a crossdependence of the presence of one 
attribute to the meaning of another attribute. Another problem appears 
if neither initialAmount nor initialConcentration is given. This can 
legally happen in a model where some species are set by rules.




The description would change as follows:

Delete everything from the start of the second paragraph
"The value can be assigned.." to and including "Case of initialAmount 
being used."

Delete paragraph "Case of initialConcentration being used...".

Add:

The units field refers to the units of substance, so if initialAmount is 
given, this number is in terms of the declared units. Concentratoins of 
the species, as used in "initialConcnetration" or in any occurence of 
the species ID in a rate law, are in terms of the declared units of the 
species devided by the declared units of the enclosing compartment.



The description of page 10 (starting with "- species identifier:"
would also simplify.
Page 23 also simplifies.



Page 21, Section 4.7.3:
Should parameters have default "dimensionless" units?

The default units for parameters are "dimensionless".



Page 21 Section 4.7.6

Change units to "molar" or "moleperliter" or change the names to 
something that does not start with "Km", since this is confusing.
Example:

<parameter id="tau1" value="2.3" units="second"/>
<parameter id="Km1" value="10.7" units="moleperliter"/>



Page 26 Section 4.9.1
I strongly oppose the deletion of the attribute "stoichoimetry".
