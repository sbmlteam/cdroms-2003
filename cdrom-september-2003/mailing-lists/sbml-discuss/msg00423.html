Now that the amount of traffic on this list has settled down just a
little, here's another working group announcement.

Parameter Sets Working Group

Purpose:
To develop a mechanism whereby parameter values can be defined separately
from the structural definition of a model (i.e., the parts of the SBML
file other than parameter values). A hierarchical structure supporting a
cascade of parameter settings with clear definition for which value in the
hierarchy for a given parameter takes precedence is desired.

Initiator: Cliff Shaffer

Inital members (as defined by who signed up in Ft. Lauderdale):
Andrew Finney
Marc Vass
Ben Bornstein
Hugh Spence
Martin Grinkel
Mike McCollum
Greg Peterson
Nick Allen
Mike Hucka

[Sorry if I mispelled any names, but some of these were hard to read from
the handwritten list.]

Outcome: Our goal is to create a proposal by the next Forum meeting in
November/December, sample models with parameter sets demonstrating the
proposal, and a reference implementation corresponding to the proposal.

Duration: No longer than necessary. We'll make a proposal for the
November/December meeting, and revise as necessary for final agreement
next May. In practice, we'll have to be done well before May because
people will long since have been implementing the revised proposal.

Starting points: Attached is a draft proposal document from Andrew Finney,
Mike Hucka, and Ben Bornstein. This describes two possible approaches, an
XSLT-based approach and a non-XSLT based approach. Marc Vass already has
an implementation for parameter sets that is very similar to the non-XSLT
proposal.

Interactions: It has already been noted that there is likely to be some
interaction between the components working group and this one. Both deal
with some of the same management issues: Multiple files, perhaps with
cascades of included subfiles.
