One thing to note about this: I don't think we can use the
same tag ('stochiometry') for both an attribute an an
element on the same parent element in XML.  So one of them
would need to be named something else.

  afinney> In a nutshell Wayne is proposing that a
  afinney> scheme where species has two mutually exclusive stoichiometry fields
  afinney> one of attribute type double, the other of element type MathML.
  afinney> FYI doubles don't have to include a period or decimal point in XML
  afinney> The attribute would default to 1.

  afinney> In this scheme the following would be valid speciesReference structures:

  afinney> <speciesReference species="foo"/>  (stoichiometery defaults to 1)

  afinney> <speciesReference species="foo" stoichiometry="2"/>

  afinney> <speciesReference species="foo" stoichiometry="2.5"/>

  afinney> <speciesReference species="foo">
  afinney> <stoichiometry>
  afinney> <math xmlns="http:/www.w3.org/1998/Math/MathML">
  afinney> <ci>k</ci>
  afinney> </math>
  afinney> </stoichiometry>
  afinney> </speciesReference>
