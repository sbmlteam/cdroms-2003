Hi Folks,

for those of you, who are interested in model composition,
here we go with the initial announcement for the
Model Composition Special Interest Group.

I propose the following as a charter, I expect this as a start of some
discussions and contributions are welcome:

Charter of the 
Model Composition Special Interest Group
at the SBML Forum

2. Discussion Forum 
   Currently we use the sbml-discuss Mailing list at
   sbml-discuss@caltech.edu with Subject: [sbml-discuss] COMP ...

   Archives are kept at:
   http://www.sbw-sbml.org/sbml-discuss/archive/maillist.html

   I will prepare some webspace (possibly for group editing) at
   http://www.mpi-magdeburg.mpg.de/zlocal/martins/sbml-comp/
   Mainly the different Proposals will be kept there intially.

3. Mission statement and Scope:
   
   Definition of an extension of the SBML standard that allows for 
   composition of models out of partial models. `Model' refers here 
   to a model as some construct containing all elements that are 
   specified in the current and future SBML standard(s).
   Main features to achieve:
   - Definition of different models in SBML documents
     as submodels or modules (submodels with specified interface)
     (see below)
   - Instantiation of submodels in a model (->model-instances)
   - Linkage of different model-instances
   - Parametrization of model-instances (in cooperation with the
       parameter-sets SIG)
   This does not necessarily mean that two random SBML models from 
   a repository that describe adjacent pathways can be composed as
   they are.
   Optional Goal: Allow for combining SBML models with CellML components.

4. Termination criterium:
   Model composition is included into the SBML Level 3

   or

   The idea is considered as nonsense for Systems-Biology and dropped
   from the SBML Agenda

   Timeframe: I expect a working proposal and first reference
   implementations for the next SBML Forum Meeting in
   Nov 2003 S. Lois 

5. Deliverables:
   - SBML standard proposal.
   - Reference implementation of reading and writing SBML+COMP in 
     a modeling tool.

6. Dependencies of other groups:
   As mentioned before: Model instances often need to be parametrized.
   Therefore we need some standard that allows for setting at least
   parameters and initial values inside model instances. This work is 
   done by the Parameter Sets group.

7. Joining the group:
   Please send a email to the Chair.


--8<--

Additional comments:
- I suggested Andrew for co-chairing since he has already done
  (2?) proposals for composion and knows the efforts across the SBML
  community very well.

- 'Submodels and Modules' or the story about defined Interfaces.
  This is of course a controversal point but:
  I would like to include both approaches. Whether the biologists will
  use this or not can not be answered from the view of a programmer.

  <IMHO> This is a question of software engineering: Large models are
  large software projects which will be tedious to maintain without 
  interfaces. But sofware engineering is learned by doing your own
  failures and experiencing the problems, not by lectures of geeks and 
  reading books </IMHO>

  We should not stop our progress with discussions of this point but 
  define something that works with or without interfaces.
  (Whatever you will do with this inside the tool finally)

- Reference implementation: I see as an initial goal that a SBML
  reader library recognizes also composition tags. This lib should
  then be used by two composition-aware tools (Promot and ???).

  I don't think that It makes sense to define a first approach inside 
  <annotation> since models will not make any sense without submodels.
  Another goal could be to write a program that reads composed models
  and flattens them out to some sort of SBML L2.

- Finally `Hello, is there anybody out there ...':
  Which tools for Sysbio besides Promot support composition and how, 
  to which extent ...
  I would like to learn more about the different concepts we 
  should support in the proposal.
