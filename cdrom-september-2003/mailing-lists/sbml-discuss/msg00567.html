This is why we chose to differentiate between layout information and render 
information in our proposal (by just ignoring the render part so far). 
For species it is easy. The layout information is the position of the glyph 
(we also added the size). We do not say whether it is a circle or a square, 
which color, ...
For the reactions it is not so easy. We compromised on what we think is a 
quite abstract representation of a reaction glyph: The points where the 
substrate arcs meet and the point where the product arcs meet. Alternatively 
you can read this as one point where all the arcs meet and the tangent 
direction. 
When we first wrote our proposal we were not shure if additional information 
about the curves really belongs to the layout information, so this 
information is not yet in there (I now believe it should be in).  

This is also why we think a species reference glyph should have a role 
attribute. A renderer that decides not to bother reading detailed curve 
information from the file should have some clue how to render. And since 
glyphs that don´t refer to a specific model object are allowed (at least in 
our proposal) the information can not be retrieved from the model.
