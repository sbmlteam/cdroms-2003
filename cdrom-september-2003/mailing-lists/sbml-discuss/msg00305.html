You know (yeah, I'm still coding L1V2 import--I'm
trying to figure out how to do the local namespace
thing in my simulator with only global namespace)
one thing that's always bothered me--and even more
so now that we've changed "specie" to "species"--is
the misnaming of collection properties like the
"parameter" property in KineticLaw, and, now with
L1V2, the "product" and "reactant" properties in
Reaction. It's *only* in the spec, since these get
turned into <listOf*> in markup. It just seems to
me the document should say, for example,

Reaction
name: SName
reactants: SpeciesReference[0..*]
products: SpeciesReference[0..*]
[yadda yadda yadda]

Again, this doesn't change anyone's code, it
ONLY changes the text in the spec, and it makes
it a lot clearer that we're talking about lists
or collections.

John
