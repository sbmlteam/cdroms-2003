I have a few questions about the Layout Extension,

In looking at the layout stored for Reactions, I see that it should be 
represented as:

<sl2:listOfReactionGRs>
      <sl2:reactionGR id="reaction_0_GR" ref="reaction_0" x1="119.0" y1="166.0"
                      x2="120.0" y2="183.0"/>

but the sample file does not include this "edge layout" information.  
Obviously, the edges are drawn based on the location of their source and 
target nodes, but I am unsure how to add in edge-bend layout 
information.  There are many time-consuming layout algorithms that are 
able to make really nice orthoganal layouts that utilize edge-bends, and 
it would be nice to store that layout.

So, if this can be done by simply adding "x3/y3" etc. to the above 
example, that would be swell.  Otherwise, let me know how this can be 
integrated.
