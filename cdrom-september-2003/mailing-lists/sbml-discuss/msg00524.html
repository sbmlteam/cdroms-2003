Kevin

If only it was that simple.

I don't disagree that this might be useful.

However getting agreement on a controlled vocabulary like this
is not straight forward.

And even if we did I don't believe that in general CVs can be considered
static especially relative to the rest of sbml so we need a mechanism
for seperating out the cvs from the rest of the language and then
maintaining the cvs and developing a process for that.

The CV working group is the context for these developments.
http://www.sbw-sbml.org/sbml-discuss/archive/msg00510.html

If you could contribute an outline CV for modifiers...

In the meantime you could investigate using GO via
CellML metadata with SBML or using an annotation.
