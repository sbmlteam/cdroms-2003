  mhucka> One thing to note about this: I don't think we can use the
  mhucka> same tag ('stochiometry') for both an attribute an an
  mhucka> element on the same parent element in XML.
  mhucka> So one of them would need to be named something else.

Hi Mike,

     Why?  Is this a SHUCS limitation or is it impossible to specify
XML Schema constraints if an attribute and subelement share the same
name?

     I know Ant (a Java build system that uses XML as its file format)
allows for this sort of "overloading".  In fact, it's used quite
(I think too) frequently.

     From both a SAX and DOM API perspective, there is no conflict
because XML attributes and elements exist in separate "namespaces".
SAX defines a notification mechanism for elements and a class that
holds attribute name / value pairs.  In DOM, elements and attributes
are represented by two separate classes (e.g., DOMNode and DOMAttr).

     If it really is a problem, then I would suggest:

       <stoichiometryMath> <math> ... </math> </stoichiometryMath>


Ben
