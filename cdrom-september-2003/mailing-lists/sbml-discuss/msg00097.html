People,

We're getting ready to issue revised drafts of SBML
specifications for review and have run into some questions
about certain matters.  I'd like to try to resolve a couple
of the questions by discussing them on this list.

The first one is: should compartments be optional in SBML?
Note that if compartments are optional, then by necessity,
the 'compartment' attribute on species definitions must also
be optional.

The arguments FOR making compartments optional in a model are:

F1) A number of tools already make the assumption that in
    the absence of specific compartment information, a
    single compartment of unit volume should be assumed for
    the entire model.

F2) The assumption of a default compartment with unit volume
    and specific units of volume can be a well-defined
    assumption, in the sense that it provides enough
    information that there is no possible ambiguity as to
    intent (provided that the default is clearly stated in
    the specification, of course).

The arguments AGAINST making compartments optional are:

A1) At the Stockholm meeting, there was a general consensus
    that SBML should not make assumptions about default
    values for any quantities, and that instead the absence
    of a value for a quantity indicates that this value is
    undefined and should be obtained by some other means.
    Defining a default compartment violates this principle.
