Charter of the SBML Arrays Working Group
----------------------------------------

Background

At the recent SBML forum meeting in Florida it was decided to
setup working groups to develop proposals for incorporation
into SBML.  Several people at the meeting signed up to
a working group to develop a proposal for arrays in SBML

Mission Statement and Scope

Definition of an extension to SBML to allow the construction
of models from homogeneous collections of objects.  Attributes
of these objects can vary systematically across these collections.

Any viable proposal must:

- support a mechanism for describing the connections between array
elements
- be compatible with other proposals in particular model composition and
dynamic structures

Main issues to resolve:

- What objects can occur in these collections?
- What is the correct mechanism for defining collections?
	for example should we consider sets, bags etc as alternatives
	to arrays.
- Do we need syntactic sugar to simplify the use of or compress the
encoding of arrays?
- Should a function for defining the sparseness of an array be dynamic
	(vary during simulation)?

Termination criterion:

   Arrays is included into the SBML Level 3

   or

   The idea is considered as nonsense for Systems-Biology and dropped
   from the SBML Agenda

   Timeframe: I expect a working proposal and first reference
   implementations for the next SBML Forum Meeting in
   Nov 2003 S. Louis 

Deliverables:

   - a Array proposal for incorporation into SBML L3
   - a reference implementation that can read and write the proposed
standard including L2 in a
     useful software context
   - at least one published model encoded in the proposal
   - optional: a tool that can translate from the proposed standard
including SBML L2 to
     unextended SBML L2.
