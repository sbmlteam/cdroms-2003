On Thursday 26 June 2003 02:17, Eric Minch wrote:
> Can someone remind me where to find "Herbert's case of flipping of behavior
> between activator and inhibitor"? I can't seem to find it, and I'm working
> on something related, so would really like to see additional examples.

I'll second that.

> In any case, it seems premature to include such functional roles as tags
> within the model. They are actually fairly high-level constructs, and I'd
> argue that they belong more in the presentation specification than in the
> model specification.

Well, that's partly true, but unlike, say, graph layout or scaling, functional 
roles are not orthogonal to the semantics of the model. I'm happy to push the 
tags into the annotation elements in order to make the loose coupling 
explicit, but it would still be nice to have a simple standard set of tags if 
they're not controversial.

> Furthermore, wherever they are eventually to be included, it would seem
> prudent to enumerate them as part of the Controlled Vocabulary effort
> rather than as ad hoc fixes in the model.

I think that depends upon how variable they will be. If everyone will wind up 
using essentially the same set of labels anyway, then use of generalized CVs 
for them would be gratuitously complex.
