Hello all,

On Tuesday 08 July 2003 19:55, Rowan Christmas wrote:
> I have a few questions about the Layout Extension,
>
> In looking at the layout stored for Reactions, I see that it should be
> represented as:
>
> <sl2:listOfReactionGRs>
>       <sl2:reactionGR id="reaction_0_GR" ref="reaction_0" x1="119.0"
> y1="166.0" x2="120.0" y2="183.0"/>
>
> but the sample file does not include this "edge layout" information.
> Obviously, the edges are drawn based on the location of their source and
> target nodes, but I am unsure how to add in edge-bend layout
> information.  There are many time-consuming layout algorithms that are
> able to make really nice orthoganal layouts that utilize edge-bends, and
> it would be nice to store that layout.
>
> So, if this can be done by simply adding "x3/y3" etc. to the above
> example, that would be swell.  Otherwise, let me know how this can be
> integrated.
>
 Indeed the information about the exact layout of the edges is not yet present 
in our suggestion. We left it out because we are not shure yet how to do it 
best. So far it is only specified which graphical objects are connected by an 
edge (this is in the speciesReferenceGR element). 
So far we have been thinking about two ways to specify the exact path of an 
edge (how it bends, but not how thick it is, what colour it has...) :
1. Just a list of additional points along the path (like Rowan mentioned)
2. a list of splines (each given by 4 points, the end point of one spline 
being the start point of the next).

This information would become part of the speciesReferenceGR object 

What do you think is the best way to specify the curves? 

Sven
