	As you probably know, I use bezier curves in jdesigner. For uniuni reaction I use just one bezier, for other reactions as use additional beziers, thus a unibi reaction uses one bzier from the uni side and two beziers to the bi side. This means that I acquire a centre point and to make things look nice the slopes at the end of the intersecting beziers match. I've thought about using other types of curve but never got round to it. As for sbml strorage, I store all control points and the centre point. Although one could compute the centre based on the centroid between all the nodes, I allow the user to move the center point, it make quite a bit of different to what users can draw.
	 
	If there are going to be different ways to draw the edges then we will proably need some information indicating what type of edge drawing it is, eg splines, beziers, straight edges with kinks etc.

		 Indeed the information about the exact layout of the edges is not yet present 
		in our suggestion. We left it out because we are not shure yet how to do it 
		best. So far it is only specified which graphical objects are connected by an 
		edge (this is in the speciesReferenceGR element). 
		So far we have been thinking about two ways to specify the exact path of an 
		edge (how it bends, but not how thick it is, what colour it has...) : 
		1. Just a list of additional points along the path (like Rowan mentioned) 
		2. a list of splines (each given by 4 points, the end point of one spline 
		being the start point of the next). 

		This information would become part of the speciesReferenceGR object
