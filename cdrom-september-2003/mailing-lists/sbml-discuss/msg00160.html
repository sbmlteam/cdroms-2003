Andrew,

You are correct - the proposal does focus on a schematic
or circuit diagram layout application.  What struck me
was that the tags defined in the proposal provide much of
what one would need for a visualization of a spatial simulation
during runtime or as a post-simulation playback.  The addition
of a third dimension to the spec would help for these types of 
application (plus the shape, color or colour :-)  
There are some other attributes of an object we might consider, but 
the list could grow indefinitely, particularly if we try to support 
molecular geometries/conformations.

One point sprung to mind when I read the schematic proposal: the
coordinates are all using 2D cartesian coordinates with an implicit
origin.  Do folks use polar/spherical coordinates at all (potentially
a help for molecular representations)?

Greg
