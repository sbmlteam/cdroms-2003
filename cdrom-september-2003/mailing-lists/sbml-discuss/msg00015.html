Hi Andrew,


> On optional parameters

My opinion is practically the same as Martin's.

SBML specification should allow model writers to choose between 
complete description where all the parameters and initial values
required to conduct the simulation is filled in, and the incomplete one
in which one or more numbers are missing.   A routine to check it may be
provided in SBML software libraries such as PySBML.

I think it should explicitly be marked undefined by using something such
as XML entity.

> On the parameter sets concept.

Generally I agree with the idea of the parameter set as a different
document type (or just a different document, if we could merge these two
schema allowing coexistence of the model and the parameter set in a
single document).


> There are some issues with this:

> a) should we allow incomplete parameter sets?
> 	IMHO yes - it becomes a useful way to support
> 	models of mutations.  We may want to allow
> 	'stacks' of parameter sets to form a simulation
> 	input.

Yes, definitely.

> b) does the model refer to the parameter set or visa-versa?
> 	certainly from a language development
> 	point of view it would be convient if 
> 	the parameter sets refer to the model.
> 	That also appears to make sense from
> 	your requirements: a model has more than 
> 	one parameter set and a parameter set is
> 	not meaningful without a model.  But a model
> 	could be considered useful without a parameter set.

No.  If we decide to have the parameter set in addition to the model
itself, we need to have one more document class to define how a
simulation session is conducted;   
i.e. it may be a sort of recipes in which it says: (1) use model A, (2)
apply parameter set B1, (3) then apply parameter set B2, (and may be (4)
run 1000 seconds, though this isn't a part of the model)...

>From the modularity view point, such model 'patching' should be done
without changing original model and parameter set documents.
To this end it should be only the session definition document (or the
recipe) which is modified.

Imagine a situation where research groups had published some parameter
sets of yeast enzyme parameters, and we have quite a few metabolic
pathway models.  If we have a reasonably good naming convention we will
be able to pick a parameter set and a model from model repository for a
simulation session.


regards,
- Kouichi
