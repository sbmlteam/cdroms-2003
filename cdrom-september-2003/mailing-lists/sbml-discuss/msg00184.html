On Wed, 21 May 2003, Michael Hucka wrote:

> A second issue is the question of whether to allow
> stoichiometry values to be expressions rather than only
> numbers.  A simple example of the utility of doing this
> arises in models using non-integer stoichiometries: the
> stoichiometry value may be the same in two of the
> 'speciesReference' structures in a reaction, and it would be
> useful to be able to define a parameter for the value and
> using that parameter instead of repeating the same number
> (because the latter is more error-prone).

I think there's another argument beyond the fact that having to put the
same number in two places is error-prone. By making these two places be
the same variable, you are expressing an intention that they in fact have
some relationship. This is pretty much the same reason why we were adamant
about permitting defined rate laws. Sure, we can get the necessary
mathematics at some physical layer by repeating the same mathematical
expressions in multiple places. But when you give a name to a concept, and
then use that name in multiple places, you are making explicit the
relationships in your model. That might let your software do more.
