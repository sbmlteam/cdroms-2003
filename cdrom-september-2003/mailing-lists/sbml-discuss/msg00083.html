OK I think you've run into a typo in the SBML Level 1 document

the text that's been corrected in the level 2 document for species is:
---
The units of the substance quantity may be explicitly set using the
optional field units. The value assigned to units must be chosen from
one of the following possibilities: one
of base unit names from Table 2 on page 13, the name "substance", or a
new unit name defined by a unit
definition in the enclosing model. If absent, the units default to the
value set by the built-in "substance"
---

> One problem is that I am not sure 
> which software 
> actually supports the use of units, and of converting between 
> quantity and 
> concentration, taking into account the volume. 

I'm not aware that any do at this stage.

What the Level 2 standard says, at the moment, is that the default units
for species attribute initialAmount
are 'substance' which is moles by default.  That's the intention of the
Level 1 document as well.

We could potentially change this default to concentration
'substance/volume' which is mole/litre
by default.  I think I'd like to know what other people think about
this.
The original idea was to enable a switch between numbers of molecules
('item') and moles for all
species simply by changing the definition of the 'substance' units.

yours Andrew
